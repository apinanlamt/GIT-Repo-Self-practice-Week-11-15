///// Week 11 //////
วันนี้ผมได้ทำโปรเจกต์เว็บ “Quote Manager” ให้สามารถใช้งานได้จริงครบทุกส่วน โดยมีจุดประสงค์เพื่อฝึกใช้ Fetch API ทำงานร่วมกับ JSON Server ในการจัดการข้อมูลแบบ CRUD (Create, Read, Update, Delete)
เริ่มจากการเตรียมโครงสร้างโปรเจกต์ที่มีอยู่แล้ว ทั้งหน้าเว็บ (index.html), การตกแต่ง (style.css), และไฟล์ฐานข้อมูล (data/db.json) จากนั้นผมไปเขียนและปรับปรุงส่วนของ JavaScript ให้สมบูรณ์ โดยเพิ่มโค้ดใน fetchUtils.js และ quoteManagement.js เพื่อให้เรียก API ได้ครบทุกคำสั่ง เช่น การเพิ่ม ลบ และแก้ไขคำคม รวมถึงการโหลดข้อมูลทั้งหมดมาแสดงบนหน้าเว็บ
ต่อมาผมได้ปรับไฟล์ app.js ให้ควบคุมส่วนของการแสดงผลและการทำงานของปุ่มต่าง ๆ เช่น ปุ่ม Add/Edit สำหรับเพิ่มหรือแก้คำคม และปุ่ม Delete สำหรับลบข้อมูลที่ไม่ต้องการ
ช่วงท้ายของวัน ผมติดตั้งและตั้งค่า JSON Server เพื่อใช้เป็น backend จำลองสำหรับเก็บข้อมูลคำคม โดยใช้คำสั่ง npm run backend ให้เซิร์ฟเวอร์ทำงานที่พอร์ต 5001 จากนั้นเปิดอีกหน้าต่างเพื่อรันเว็บฝั่ง frontend ด้วย npm run dev เมื่อเปิดเว็บในเบราว์เซอร์ ทุกอย่างก็ทำงานได้จริง — ข้อมูลโหลดขึ้นมา, เพิ่ม ลบ แก้ไข ได้ครบถ้วน

ประโยชน์ที่จากการฝึก Practice
วันนี้ผมได้เรียนรู้และทำครบทุกขั้นตอน ตั้งแต่การเชื่อมต่อ front-end กับ back-end ไปจนถึงการทดสอบระบบจริง เว็บ Quote Manager จึงเสร็จสมบูรณ์พร้อมใช้งานในวันนี้

///// Week 12 – Input Masking & Multi-page Form //////
สัปดาห์นี้ผมได้ทำโปรเจกต์ Input Masking เพื่อฝึกการจัดการ Event Input แบบละเอียด รวมถึงการประมวลผลข้อมูลก่อนแสดงผลให้ผู้ใช้ จุดประสงค์คือต้องการให้ผู้ใช้กรอกข้อมูลได้ง่าย ถูกต้อง และมีรูปแบบตามมาตรฐานทันทีที่พิมพ์
เริ่มจากสร้างหน้า index.html สำหรับกรอกเบอร์โทรและบัตรประชาชน จากนั้นผมเขียนฟังก์ชัน Masking ในไฟล์ mask.js เพื่อจัดรูปแบบข้อมูลอัตโนมัติ เช่น แปลงเป็น xxx-xxx-xxxx สำหรับเบอร์โทรและ x-xxxx-xxxxx-xx-x สำหรับบัตรประชาชน
ต่อมาผมเพิ่มระบบ Submit โดยเมื่อผู้ใช้กดปุ่ม ระบบจะส่งข้อมูลผ่าน URL Query ไปยังหน้า result.html ซึ่งแสดงข้อมูลที่จัดรูปแบบเรียบร้อยแล้ว ทำให้โปรเจกต์นี้เป็นเว็บสองหน้าที่เชื่อมโยงข้อมูลกันครบวงจร

ประโยชน์ที่ได้จากการฝึก Practice
วันนี้ผมได้รู้จักการทำ Input Masking แบบ manual โดยไม่ใช้ไลบรารี รวมถึงการทำงานกับ Query String และ multi-page interaction ซึ่งเป็นพื้นฐานของระบบฟอร์มจริงที่ใช้ในงานธุรกิจ เช่น สมัครสมาชิก, กรอกข้อมูลลูกค้า หรือระบบจองต่าง ๆ

////// Week 13 – Advanced DOM & Event Delegation //////
วันนี้ผมได้ฝึกทำโปรเจกต์เกี่ยวกับ DOM ขั้นสูง โดยเน้นการจัดการ Event Bubbling และ Event Delegation บนโครงสร้างที่ซับซ้อนมากขึ้น จุดประสงค์คือเพื่อให้สามารถควบคุมเหตุการณ์ของ element จำนวนมากโดยไม่ต้องผูก event ทีละตัว ซึ่งช่วยลดโค้ดซ้ำและเพิ่มประสิทธิภาพเว็บในกรณีที่มีการสร้าง element แบบไดนามิก
เริ่มจากการสร้างหน้าเว็บที่มีรายการ To-Do หลายระดับ (nested list) พร้อมปุ่ม Add, Edit, Delete ในแต่ละระดับ จากนั้นผมเพิ่ม input form เพื่อให้ผู้ใช้สามารถเพิ่มรายการใหม่ได้ไม่จำกัด เมื่อเพิ่มหรือแก้ไขข้อมูล DOM ก็จะอัปเดตแบบเรียลไทม์โดยใช้ Delegation ทั้งชุด ทำให้ควบคุมเหตุการณ์ทั้งหมดด้วย event listener เพียงตัวเดียว
ในการฝึกครั้งนี้ผมได้จัดการทั้งส่วนของการสร้าง element, การลบ node, การแก้ไขข้อมูลในรายการ รวมถึงการตรวจจับเหตุการณ์ที่เกิดจาก element ที่ถูกสร้างขึ้นใหม่ใน runtime ซึ่งเป็นทักษะที่จำเป็นมากสำหรับ front-end development ในเว็บสมัยใหม่

ประโยชน์ที่ได้จากการฝึก Practice
วันนี้ผมเข้าใจโครงสร้าง DOM ลึกขึ้น รวมถึงวิธีใช้ Event Bubbling/Delegation เพื่อให้เว็บรองรับ UI ที่ซับซ้อนและ dynamic มาก ๆ โค้ดที่เขียนอ่านง่ายขึ้น และสามารถต่อยอดไปทำ component-interaction แบบมืออาชีพได้ในอนาคต

////// Week 14 – Fetch API Project (Movie/Quote Search) //////
วันนี้ผมได้ฝึกโปรเจกต์ที่เน้นการใช้ Fetch API เพื่อสื่อสารกับ REST API จริง จุดประสงค์คือเพื่อให้บริหารจัดการข้อมูลจาก backend ได้คล่องขึ้น และเข้าใจ lifecycle ของ request/response แบบ async
เริ่มจากสร้างหน้าเว็บ โดยให้ผู้ใช้กรอกคำค้น เพื่อเรียก API แบบ asynchronous พร้อมรองรับสถานะ loading และ error handling เช่น เมื่อ API ล่มหรือไม่พบข้อมูล
เมื่อข้อมูลถูกดึงมา เพื่อสร้าง element แบบ dynamic 

ประโยชน์ที่ได้จากการฝึก Practice
วันนี้ผมเข้าใจการทำงานแบบ asynchronous และการเรียก API ภายนอกแบบมีโครงสร้างมากขึ้น สามารถสร้างเว็บที่ดึงข้อมูลสดจากเซิร์ฟเวอร์ได้จริง ทำให้พร้อมต่อยอดไปสู่โปรเจกต์ใหญ่ เช่น Dashboard หรือระบบจัดการข้อมูลเต็มรูปแบบ

////// Week 15 – DateTime & New Year Countdown ////// 
ในสัปดาห์นี้ผมได้ทำโปรเจกต์เกี่ยวกับ Date/Time โดยสร้างระบบนับถอยหลังปีใหม่แบบเรียลไทม์ จุดประสงค์คือเพื่อฝึกการคำนวณเวลาระหว่างปัจจุบันและวันที่กำหนดโดยใช้ JavaScript Date API
เริ่มจากสร้างหน้าเว็บ countdown.html พร้อมพื้นที่แสดงวัน ชั่วโมง นาที วินาที จากนั้นผมไปเขียนโค้ดใน countdown.js เพื่อคำนวณส่วนต่างของเวลาแล้วอัปเดตค่าทุกวินาทีด้วย setInterval
โครงการนี้ยังรวมถึงการจัดรูปแบบเวลา เช่น เติมเลขศูนย์ (00–09) และจัดการกรณี countdown หมดเวลาพอดีเพื่อแสดงข้อความ “Happy New Year!” พร้อมแอนิเมชันเล็ก ๆ ที่ผมใส่เพิ่มเข้าไปใน CSS

ประโยชน์ที่ได้จากการฝึก Practice
วันนี้ผมได้เข้าใจการทำงานของ Date, Timestamp, Time difference และการอัปเดต UI แบบ real-time ซึ่งเป็นทักษะสำคัญสำหรับการทำระบบที่ต้องใช้เวลา เช่น การจอง, การ Flash Sale, ระบบลงทะเบียน, หรือ event countdown ในหน้าเว็บจริง
